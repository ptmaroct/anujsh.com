Linux gaming is faster because Windows APIs are becoming Linux kernel features

The headline isn't that Proton got better — it's that Microsoft's job of maintaining its own APIs is now being done, more efficiently, by someone else's kernel.

For a decade the Linux gaming pitch went: it's almost as good, give it a chance. The thing that has flipped this year is not driver maturity but a quieter structural fact — large parts of the Win32 and DirectX surface area are now implemented directly inside the Linux kernel, against the kernel's own scheduler and memory model, rather than through translation in user space.

Once that happens, the so-called compatibility layer stops being a tax and starts being a head start. Linux gets the API behaviour the games were designed against, plus a lighter graphics stack and an OS that doesn't fight you for cycles. Microsoft is now in the awkward position of competing on Windows feature parity against a kernel that ships every six weeks and doesn't carry twenty years of telemetry.
